Detailed Description

Actually rendering a view is going to be tricky, and probably unnecessary. For testing purposes, the caller needs to be able to assert the view name and the provided data. This fake renderer will execute to arrive at:

  1. the response body will be the view name
  2. The http context on the action context will have the following Items set:
    • Model
    • ViewData
    • ContentType
    • StatusCode
